Programming Assignment 1

Due Date - Sept 30, 2019

Overview


In this project, you are going to implement a client and a server program that sends a command over the network. The command simply turns on a LED light. However, the LED light is connected to a server and it speaks a special protocol that we don’t know. All we know is the packet format used for turning on the light. Your job is to create this packet and then send it to the server for turning on this light.
The objective is to familiarize you with the concept of sockets.


Server Specifications

The server takes two arguments:

$ lightserver <PORT><LOG FILE LOCATION>

1.PORT - The port server listens on.

2.Log file location - Where you will keep record of commands you received.

For example:
$ lightserver 6543 /tmp

If your server receives a command that is not supported, it simply logs it as UNKNOWN COMMAND: COMMAND NAME and ignores it.

If your server receives a command that is supported, it logs it as EXECUTING SUPPORTED COMMAND: COMMAND NAME. TODO add executable for turning on the light


Server Requirements
The Server must:

  1. Listen to commands indefinitely. It must not exit after receiving a single command (5 pts)
  2. Parse the packet and logs the command (15pts)
  3. Interface with the light control on supported command (5pts)
  4. Print an error message if receives a command it does not understand (5 pts)

  5. Reply to the Client with REPLY message (10pts)

    |MESSAGE TYPE - 17 (Interger)| MESSAGE (OK|FAILED) - 1 BYTE|


Client Specifications 

$ lightclient <SERVER-IP><PORT>

The client also takes two arguments:

1.Server IP - The IP address of the server.

2.PORT - The port the server listens on.

For example:
$ lightclient 192.168.2.1 6543

Client Requirements
The Client must:

  1. Send out a packet in the following format:

    |MESSAGE TYPE - 17 (Interger)| MESSAGE (LIGHTON) - 1 BYTE|



The Client must:

  1. Format the packet and send it over the wire (10 pts)
  2. Wait for reply until it times out (timeout = 10 second) (10pts)
  3. Exit with an error message if it can not connect to the server or is supplied with wrong port/IP (10pts)
  4. Terminate the connection properly is OK message is received. It also prints “SUCCESS, CONNECTION TERMINATING” (10pts)
  5. Retries once if FAILED message is received, it also prints “FAILURE, RETRY” (10pts)
  6. Exits is a FAILED message is received again, it also prints “FAILURE: GIVING UP” (10pts)
